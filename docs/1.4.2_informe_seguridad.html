1. Objetivo y alcance

Este informe documenta las pruebas de seguridad realizadas sobre el sistema de e-commerce Wooden House, así como las medidas de protección implementadas. El análisis cubre: autenticación, autorización, validación de datos, protección contra inyecciones, manejo de sesiones, cabeceras de seguridad HTTP, seguridad en pagos y protección de archivos.

Las pruebas se alinean con el estándar OWASP Top 10 (2021) y se realizaron de forma manual en el entorno de producción utilizando herramientas disponibles públicamente.

6. Seguridad en pagos (PCI DSS)

El módulo de pagos sigue el estándar PCI DSS SAQ A (el nivel más básico y común para e-commerce que no procesa directamente datos de tarjeta):

  • No almacenamiento de datos de tarjeta: Los datos de tarjeta se ingresan directamente en el iframe seguro de Stripe. El servidor de Wooden House nunca recibe ni almacena números de tarjeta, fechas de expiración ni CVV.
  • Tokenización: Stripe genera un PaymentIntent ID que se guarda en la BD como referencia. Este ID no puede usarse para realizar cargos adicionales.
  • Cifrado en tránsito: Toda comunicación con Stripe y PayPal ocurre sobre HTTPS/TLS.
  • Verificación de webhooks: Firma HMAC verificada antes de actualizar estado de pagos.
  • Modo live / sandbox: Las pruebas se realizaron en modo sandbox de ambas plataformas sin usar tarjetas reales.

7. Reglas de seguridad de Firebase

Firestore Rules

Las reglas de Firestore permiten leer y escribir notificaciones únicamente a usuarios autenticados con rol de personal interno (usuarios_personal). Los clientes no tienen acceso a Firestore.

Storage Rules

Solo usuarios autenticados pueden subir imágenes a Firebase Storage. Las imágenes son públicas de solo lectura (para el catálogo) pero solo personal interno puede escribir.

8. Conclusiones

El sistema de e-commerce Wooden House implementa medidas de seguridad sólidas en todas las capas del sistema. De los 28 controles evaluados, 27 están completamente mitigados y 1 fue calificado como parcial (log de auditoría de acciones del personal, fuera del alcance del proyecto).

El sitio obtuvo una calificación de A / A+ en SecurityHeaders.com, lo que indica una configuración correcta de las cabeceras de seguridad HTTP.

No se encontraron vulnerabilidades críticas. El sistema está protegido contra las amenazas más comunes del OWASP Top 10 mediante una combinación de:

  • PDO con consultas preparadas (anti SQL injection)
  • Sanitización en dos capas — cliente y servidor (anti XSS)
  • Autenticación robusta con Firebase Auth + verificación JWT en backend
  • Cabeceras de seguridad HTTP completas (CSP, HSTS, X-Frame-Options, etc.)
  • Protección de datos de pago mediante tokenización (Stripe/PayPal)
  • Verificación de webhooks con firmas digitales
  • Rate limiting y honeypot contra bots
  • Control de acceso por roles implementado en frontend y backend
